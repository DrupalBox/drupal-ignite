Drupal CI. Today.

From Zero to Continuous Integration in the blink of an eye.

Drupal Ignite is a tool that helps you setting up a Drupal working environment finely-tuned for Continuous Integration, testing and Code Driven Development. It takes a strongly opinionated view on the toolset and development flow in order to provide an enterprise-class standard for developing with Drupal.

Getting Started Documentation

Getting started

1. Installation

You can either download the zip or clone the repository from github and place it in a directory of your choice, eg: /home/foo/documents/code/drupal/drupal-ignite-core/

2. Ignition

Once you have drupal ignite directory set on your file system, open a terminal into it and run ./setup.sh --help to see the available options.

When you're ready, you can run setup.sh to perform the installation of the drupal ignite template in a directory of your choice (usually the web root, such as /var/www/yoursite)

3. Development

After running Drupal Ignite's setup script, you'll have a basic drupal folder where you can start developing your website, so probably you'll only need to run git add -A and git commit -m "Initial commit" and you'll be good to go!

4. Mastering

You're good to go now, but you might want to look at the introduction and the other sections to get a better grasp on how to make all the parts working together.

Composer

The PHP de-facto standard for dependency-management. Drupal Ignite makes use of it for installing some 3rd-party tools that live outside of Drupal's world, such as PHPUnit and Behat.

Drush

Also called "the Drupal's Swiss knife", Drush(aka DRUpal SHell) is a command-line tool that simplifies a whole range of Drupal-related tasks such as installing a module, backing up a site or installing one.

A Drupal module that dumps a site's configuration found in the databse to module files. It enables developers to version most parts of the website and to replicate them easily across environments.
